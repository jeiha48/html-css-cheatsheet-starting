HTML Reference Sheet:

Semantic HTML

Table Tags

Tags Name Description
<header> Header Represents introductory content at the beginning of the page, typically a group of
introductory or navigational aids
<nav> Navigation Represents a section of the page whose purpose os to provide navigational links
exampls of navigational links are menus, table of contents and indexes
<main> Main Represents the dominant content of the <body> of a document.It consists of
content directly related to the central topic of the document
<footer> Footer Represents information about the author of the section, copyright data or links to
related documents, It is usually found at the bottom of the page
<section> Section Represents a generic standalone section of a document, which doesn't have a more
specific semantic element to represent it. Sections should always have a heading, with very
few exceptions.
<article> Article Represents a self-contained composition in a document, page, application, or site,
which is intended to be independently distributable or reusable
<aside> Aside Represents a portion of a document whose content is only indirectly related to the
document's main content. Asides are frequently presented as sidebars or call-out boxes.
